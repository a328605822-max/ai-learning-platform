## 1. 核心概念讲解

### 1.1 从「一问一答」到「多轮循环」

上周的 Function Calling：用户问 → LLM 调一次工具 → 返回结果。这只能处理简单任务。

但真实世界的任务往往需要多步操作：

```
用户：「帮我查一下北京和上海今天哪个城市更暖和，然后把结果发到我的邮箱」

这需要：
Step 1: get_weather("北京")  → "25°C"
Step 2: get_weather("上海")  → "28°C"
Step 3: 比较 → "上海更暖和"
Step 4: send_email(to="user@email.com", body="上海28°C > 北京25°C")
```

**Agent 循环就是让 AI 能自己规划、执行、验证这多个步骤。**

### 1.2 ReAct 范式：推理 + 行动的交错

ReAct = **Re**asoning + **Act**ing。这是目前最主流的 Agent 范式。

### 1.3 Chain of Thought（思维链）

让 AI「把思考过程说出来」，决策质量会大幅提升。就像做数学题要写步骤——不是直接蹦出答案。

```
❌ 不好的思考：
「调用 get_weather」

✅ 好的思考（思维链）：
「用户想知道北京和上海哪个更暖和。
 我需要先获取两个城市的天气。
 先查北京 → 再查上海 → 对比温度 → 得出结论。
 第1步：call get_weather('北京')」
```

<TipBox title="一句话记住" icon={Lightbulb}>
  ReAct = 让 AI 像人类一样工作：先想清楚，再动手，看了结果，再想下一步。思维链把这个「想」的过程显式化。
</TipBox>

## 2. 动手实践

### 升级 Agent：支持多步循环

基于上周的 `agent.py`，升级为完整的多步 Agent：

```python
# agent_v2.py
import json
import os
from dotenv import load_dotenv
from anthropic import Anthropic
from tools import TOOLS
from tool_executor import execute_tool

load_dotenv()
client = Anthropic(api_key=os.getenv("CLAUDE_API_KEY"))

SYSTEM_PROMPT = """你是一个 AI Agent，可以多次使用工具来完成复杂任务。

## 工作方式（ReAct 模式）
1. 思考：分析当前情况，决定下一步做什么
2. 行动：调用合适的工具
3. 观察：分析工具返回的结果
4. 重复：如果任务未完成，继续第1步

## 规则
- 如果一次工具调用就能完成任务，直接调一个工具后给出最终回答
- 如果需要多次调用，按顺序一步步来
- 每次只做一件事，看看结果再决定下一步
- 当掌握足够信息时，给出完整的最终回答"""

def run_agent_v2(user_task: str, max_steps: int = 10, verbose: bool = True):
    """升级版 Agent —— 支持多步循环"""

    messages = [{"role": "user", "content": user_task}]
    tool_call_history = []

    for step in range(max_steps):
        if verbose:
            print(f"\n{'='*50}")
            print(f"🔄 Step {step + 1}/{max_steps}")
            print(f"{'='*50}")

        response = client.messages.create(
            model="claude-haiku-4-5-20251001",
            max_tokens=500,
            system=SYSTEM_PROMPT,
            tools=TOOLS,
            messages=messages,
        )

        if response.stop_reason == "tool_use":
            for block in response.content:
                if block.type == "tool_use":
                    tool_name = block.name
                    tool_input = block.input

                    if verbose:
                        print(f"  🤔 思考后决定：{tool_name}")
                        print(f"  📥 参数：{json.dumps(tool_input, ensure_ascii=False)}")

                    result = execute_tool(tool_name, tool_input)
                    tool_call_history.append((tool_name, tool_input, result))

                    if verbose:
                        print(f"  📤 结果：{result[:100]}...")

                    messages.append({"role": "assistant", "content": response.content})
                    messages.append({
                        "role": "user",
                        "content": [{"type": "tool_result", "tool_use_id": block.id, "content": result}]
                    })
        else:
            final_answer = response.content[0].text
            if verbose:
                print(f"\n  ✅ 任务完成！")
                print(f"  💬 最终回答：{final_answer}")
                print(f"  📊 共调用 {len(tool_call_history)} 次工具")

            return {
                "answer": final_answer,
                "steps": step + 1,
                "tool_calls": tool_call_history,
            }

    return {"answer": "达到最大步数限制", "steps": max_steps, "tool_calls": tool_call_history}


# 测试多步任务
if __name__ == "__main__":
    print("测试 1: 单步任务")
    run_agent_v2("北京今天天气怎么样？")

    print("\n\n测试 2: 多步任务")
    run_agent_v2("帮我查一下北京和上海的温度，然后告诉我哪个更暖和")

    print("\n\n测试 3: 复杂多步任务")
    run_agent_v2("北京昨天的气温是多少度？并且帮我算 123 * 456")
```

<SuccessBox title="如果你跑通了多步 Agent">
  恭喜！你已经实现了 Agent 最核心的能力——多步自主决策。你现在拥有的东西，本质上和 Claude Code、Cursor Agent、Devin 这些产品背后的技术是一样的。它们的区别在于工具更多（文件系统、终端、浏览器）、System Prompt 更精细、安全机制更完善——但底层逻辑一模一样。
</SuccessBox>

<WarningBox title="常见踩坑">
  - **Agent 会「死循环」**：LLM 有时候会重复调同一个工具。设置 `max_steps` 上限，并在 System Prompt 中明确写「不要重复调用同一个工具」
  - **上下文超长**：多轮循环后对话历史很长。Agent 的每一步都会累积到 message 数组里，注意上下文窗口上限
  - **LLM 不知道何时该停**：在 System Prompt 中明确「当你掌握足够信息来回答用户问题时，直接给出最终回答，不要再调用工具」
</WarningBox>

## 3. 三视角复盘

<ReviewPanel type="principle">

### 原理视角

Agent 循环的本质是一个「决策→执行→观测」的控制回路。这和自动控制理论中的反馈控制系统完全一致。ReAct 范式把这个回路显式化了——每一步都记录推理和行动，这让 Agent 的行为可解释、可调试。

</ReviewPanel>

<ReviewPanel type="money">

### 变现视角

多步 Agent 的价值比单步 Function Calling 高一个数量级。一个能做「数据收集→分析→报表生成→邮件发送」完整流程的 Agent，可以替代一个初级员工的日常工作。这就是为什么「AI 员工」这个概念在 2025 年如此火热。

</ReviewPanel>

<ReviewPanel type="education">

### 教育视角

**怎么给孩子讲 Agent 循环？**

「你玩游戏的时候，是不是这样：想一下下一步该怎么走（思考）→ 操作手柄（行动）→ 看看游戏发生了什么（观察）→ 再想下一步？AI Agent 就是在玩一个角色扮演游戏，每次想一小步、做一小步、看结果、继续想。一步登天的事，连 AI 都做不到。」

</ReviewPanel>
