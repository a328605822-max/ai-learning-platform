## 2. LLM 到底是怎么工作的？

这是整个模块一最核心的认知。理解了这个，你就超越了 90% 的 AI 用户。

### 一个简单的实验

想象这样一个场景：

<KeyPoint emoji="🧠">
  给你一句话：「今天天气真___」
  你会猜什么？大概率是「好」或者「差」或者「热」。
  LLM 做的事情本质上和你一模一样。
</KeyPoint>

唯一的区别是：
- 它读过的文字是你的几百万倍（整个互联网的文本）
- 它的「猜」不是随便猜，而是基于学到的海量模式，计算每个可能的下一个字的概率
- 它猜完之后，把猜出来的字加进去，继续猜下一个字，如此往复

这个过程叫做 **自回归生成（Autoregressive Generation）**。每一个新生成的词，都依赖于前面所有的词。

<TipBox title="一个重要的认知转变" icon={Lightbulb}>
  LLM 不「知道」任何事。它没有知识库，没有信念，没有理解。它只有一个能力：**给定上文，预测下文**。

  它回答「1+1=2」不是因为它懂数学，而是因为它在训练数据里见过几百万遍「1+1=2」这个模式。如果训练数据里都是「1+1=3」，它也会坚定地告诉你「1+1=3」。
</TipBox>

### 第二步：第一个 API 调用

创建一个 `.env` 文件（注意：没有文件名，只有后缀），填入你选择的 API Key：

```bash
# 选择其中一个服务商即可
OPENAI_API_KEY=sk-xxx-你的key
# DEEPSEEK_API_KEY=sk-xxx-你的key
# CLAUDE_API_KEY=sk-ant-xxx-你的key
# GLM_API_KEY=xxx-你的key
```

<InfoCallout icon={Info}>
  `.env` 文件是存放敏感信息（API Key、密码等）的标准方式。`.gitignore` 里一定要加上 `.env`，防止泄露到 GitHub。
</InfoCallout>

<ConceptCard
  title="OpenAI 调用方式"
  icon={Zap}
  explanation="最通用的方案。DeepSeek 也兼容此接口（只需改 base_url）。"
  color="blue"
/>

```python
# test_api_openai.py — 适用于 OpenAI / DeepSeek（兼容接口）
import os
from dotenv import load_dotenv
from openai import OpenAI

load_dotenv()

# OpenAI
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

# DeepSeek（取消下面注释，注释上面那行）
# client = OpenAI(
#     api_key=os.getenv("DEEPSEEK_API_KEY"),
#     base_url="https://api.deepseek.com"
# )

response = client.chat.completions.create(
    model="gpt-4o-mini",  # DeepSeek 用 "deepseek-chat"
    messages=[{"role": "user", "content": "用一句话解释什么是 AI"}]
)

print(response.choices[0].message.content)
```

<ConceptCard
  title="Anthropic (Claude) 调用方式"
  icon={Brain}
  explanation="Claude API，原生支持 Tool Use，适合 Agent 开发。"
  color="purple"
/>

```python
# test_api_claude.py
import os
from dotenv import load_dotenv
from anthropic import Anthropic

load_dotenv()

client = Anthropic(api_key=os.getenv("CLAUDE_API_KEY"))

message = client.messages.create(
    model="claude-haiku-4-5-20251001",
    max_tokens=200,
    messages=[{"role": "user", "content": "用一句话解释什么是 AI"}]
)

print(message.content[0].text)
```

<ConceptCard
  title="智谱 GLM 调用方式"
  icon={Sparkles}
  explanation="国产大模型，中文能力强，价格实惠。"

```python
# test_api_glm.py — 智谱 GLM（OpenAI 兼容模式）
import os
from dotenv import load_dotenv
from openai import OpenAI

load_dotenv()

client = OpenAI(
    api_key=os.getenv("GLM_API_KEY"),
    base_url="https://open.bigmodel.cn/api/paas/v4/"
)

response = client.chat.completions.create(
    model="glm-4-flash",  # 免费/便宜的模型
    messages=[{"role": "user", "content": "用一句话解释什么是 AI"}]
)

print(response.choices[0].message.content)
```

<WarningBox title="常见踩坑">
  - **401 错误**：API Key 填错了或已过期。去后台检查。
  - **429 错误**：调用太频繁或余额不足，等几秒再试。
  - **model not found**：模型名称拼写错误。复制粘贴别手打。
  - **.env 文件没生效**：确保 `.env` 文件和 `test_api.py` 在同一目录。
  - **DeepSeek/GLM 调用报 404**：检查 `base_url` 是否正确（末尾带 `/`）。
  - **API 调用超时**：OpenAI/Claude 国内访问可能需要代理，推荐用 DeepSeek 或智谱 GLM。
</WarningBox>

## 6. 三视角复盘

<ReviewPanel type="principle">
  **我这周搞懂的核心概念**：LLM 的本质是「猜下一个字」。它不「知道」任何事，它只是在海量训练数据中学到了「什么字通常在什么字后面出现」的模式。

  **我能用一句话说清吗？** AI 就像一个读过整个互联网、记忆力超强的鹦鹉——它极其擅长猜你下一句想说什么，但它不知道自己在说什么。
</ReviewPanel>

<ReviewPanel type="money">
  **这个技术能解决谁的什么问题？** 任何需要「生成文字」的场景：写邮件、写报告、写代码、写文案、翻译、总结、回答问题。市场上已经有靠「套壳 ChatGPT API + 垂直场景 prompt」月入过万的产品。

  **有人愿意为此付费吗？** 纯通用聊天产品很难收费（ChatGPT 免费）。但如果你把 API 包装成「特定场景的工具」——比如专门写小红书文案的——就有人愿意付费。用户买的不是 AI，是「省时间」。
</ReviewPanel>

<ReviewPanel type="education">
  **怎么给孩子讲？**
  「宝宝学说话的时候，是不是听爸爸妈妈说了很多很多话，然后慢慢就会说了？AI 也是一样的——它'听'（读）了整个互联网上所有的文字，然后学会了怎么说话。但是呢，它不知道自己说的是什么，它只是在模仿。就像你小时候可能会跟着电视说广告词，但不知道广告词是什么意思。」
</ReviewPanel>
