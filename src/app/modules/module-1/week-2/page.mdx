## 1. 核心概念讲解

### 1.1 Token：AI 的「最小阅读单位」

人读的是「字」，AI 读的是 **Token**。一个 Token 可能是：

- 一个完整的常用词（如 "the"、"我"、"是"）
- 一个词的一部分（如 "playing" → "play" + "ing"）
- 一个标点符号
- 一个空格

**关键事实：中文和英文的 Token 效率完全不同。**

### 为什么 Token 很重要？

1. **计费**：API 按 Token 数计费。你每次对话的输入和输出都会消耗 Token。
2. **上下文窗口限制**：每个模型一次最多处理 N 个 Token。超出就直接「失忆」。
3. **性能**：Token 越多，处理时间越长，成本越高。

### 1.2 Context Window：AI 的「短期记忆」

想象你在和一个朋友聊天：
- 聊了 5 分钟，他记得你们在说什么 → 正常
- 聊了 2 小时，他忘了你开头说过什么 → 正常的短期记忆
- 聊了一整天，他连你叫什么名字都忘了 → 这超出了他的记忆容量

**AI 的上下文窗口（Context Window）就是它的短期记忆容量。**

**超出上下文窗口会怎样？**

AI 会「忘记」对话开头的内容。在 API 实现中，每次调用都需要把整个对话历史传回去。当对话历史超过上下文窗口，你就必须截断最前面那部分。这就是为什么长对话中 AI 会越来越「健忘」。

```python
# 上下文窗口的直观理解
context_window = [
    {"role": "user", "content": "你好"},           # 最早的消息
    {"role": "assistant", "content": "你好！"},
    {"role": "user", "content": "帮我写一首诗"},
    {"role": "assistant", "content": "春眠不觉晓..."},
    # ... 几百轮对话之后 ...
    {"role": "user", "content": "刚才我说了什么？"},  # AI 可能已经忘了
]
# 如果 token 总数超过上下文窗口，就得删掉最早的那些消息
```

### 1.3 Temperature：AI 的「创造性旋钮」

Temperature（温度）控制 AI 输出是「保守」还是「有创意」。

```
Temperature = 0.0  →  极其确定，像考试答卷
                      每次问同样的问题，答案几乎一样
                      适合：数学计算、代码生成、事实问答

Temperature = 0.7  →  平衡，最常用的设置
                      有一定变化但不会乱来

Temperature = 1.0  →  很有创意，像喝了两杯酒
                      每次答案都可能不一样
                      适合：创意写作、头脑风暴
                      
Temperature = 1.5  →  开始胡言乱语（不推荐）
```

**技术原理**：Temperature 调整的是概率分布的「平滑度」。低温度让高概率的词更高（所以更确定），高温度让概率更均匀（所以更随机）。

<TipBox title="实用建议">
  对于大多数应用场景，temperature 设置在 0.3-0.7 之间效果最好。写代码用 0-0.2，写文章用 0.6-0.8，做客服用 0.1-0.3。
</TipBox>

## 2. 动手实践

### 2.1 安装 Streamlit 并创建界面框架

如果你还没安装 Streamlit：

```bash
pip install streamlit
```

创建一个新文件 `app.py`：

```python
import streamlit as st

st.set_page_config(page_title="我的 AI 助手", page_icon="🤖")

st.title("🤖 我的 AI 助手")

# 侧边栏：参数设置
with st.sidebar:
    st.header("⚙️ 参数设置")
    
    # System Prompt
    system_prompt = st.text_area(
        "System Prompt（AI 的人设）",
        value="你是一个友好的 AI 助手，乐于帮助用户解决问题。",
        height=100
    )
    
    # Temperature 滑块
    temperature = st.slider(
        "Temperature（创造性）",
        min_value=0.0,
        max_value=1.0,
        value=0.7,
        step=0.1,
        help="0 = 确定保守，1 = 创意随机"
    )
    
    # Max tokens
    max_tokens = st.slider(
        "最大回复长度（tokens）",
        min_value=50,
        max_value=2000,
        value=500,
        step=50
    )
    
    # 显示当前设置
    st.divider()
    st.caption(f"当前温度：{temperature}")
    st.caption(f"最大长度：{max_tokens} tokens")

# 对话历史管理
if "messages" not in st.session_state:
    st.session_state.messages = []

# 显示历史消息
for msg in st.session_state.messages:
    with st.chat_message(msg["role"]):
        st.markdown(msg["content"])

# 用户输入
if prompt := st.chat_input("输入你的问题..."):
    # 添加用户消息
    st.session_state.messages.append({"role": "user", "content": prompt})
    with st.chat_message("user"):
        st.markdown(prompt)
    
    # TODO: 下周接入 API
    with st.chat_message("assistant"):
        st.markdown(f"*（这里将显示 AI 的回复。当前设置：temperature={temperature}）*")
        st.caption(f"System Prompt: {system_prompt[:50]}...")
```

运行：

```bash
streamlit run app.py
```

浏览器应该自动打开 `http://localhost:8501`，你会看到你的第一个 AI 聊天界面！

### 2.2 试试 Token 计数

创建一个 `token_counter.py`，体验 Token 是怎么计数的：

```python
# 如果你用的是 OpenAI
# pip install tiktoken
# import tiktoken

# 这里我们用一个简单的估算方法
def estimate_tokens(text: str) -> int:
    """粗略估算 token 数（中文一个字约 1-1.5 token，英文一个单词约 1.3 token）"""
    chinese_chars = sum(1 for c in text if '一' <= c <= '鿿')
    english_words = len([w for w in text.split() if w.isascii() and w.isalpha()])
    return int(chinese_chars * 1.2 + english_words * 1.3)

# 测试
test_texts = [
    "人工智能改变世界",
    "Artificial Intelligence is changing the world",
    "AI 正在改变我们的生活方式和工作方式",
]

for text in test_texts:
    tokens = estimate_tokens(text)
    print(f"原文：{text}")
    print(f"估算 Token 数：{tokens}")
    print(f"估算成本（Haiku）：${tokens * 0.25 / 1000000:.6f}")
    print()
```

## 3. 三视角复盘

<ReviewBox type="principle">

### 原理视角

**我这周搞懂的核心概念**：Token 和上下文窗口。中文的 Token 效率比英文低（更贵），但中文更紧凑（一个字能表达更多意思）。上下文窗口限制了 AI 的「记忆时长」，这是 API 调用的硬约束。Temperature 是控制 AI「靠谱程度」的最简单方法。

**我能用一句话说清吗？** Token 是 AI 的「阅读单位」，上下文窗口是 AI 的「短期记忆容量」，Temperature 是 AI 的「理性/感性旋钮」。

</ReviewBox>

<ReviewBox type="money">

### 变现视角

**成本是产品设计的硬约束。** 你设计产品时必须算清楚：每个用户的平均 Token 消耗 × 用户数 = 你的 API 成本。如果用户付你 10 元/月，但每月消耗 20 元的 API 费用，你的商业模式就不成立。**做 AI 产品的第一课不是写代码，是算成本。**

降低成本的常见思路：
- 用便宜的模型处理简单任务（Haiku）
- 用 System Prompt 限制 AI 输出长度
- 缓存常见问题的回答
- 设置用户使用频率上限

</ReviewBox>

<ReviewBox type="education">

### 教育视角

**怎么给孩子讲 Token？**
「你说话是一个字一个字说的对不对？AI 不一样，它是用'词块'来读的。比如你跟 AI 说'你好'，AI 看到的不是两个字，而是一个词块（token）。这个很像我教你认字的时候——你一开始是一个字一个字认，后来就能一眼看出一整个词了。AI 就是永远停留在了'一眼看一个词块'的阶段。」

**怎么给孩子讲上下文窗口？**
「你有没有这样的经历——老师讲了很长很长的故事，讲到后面你忘了前面讲了什么？AI 也有这个问题。它的'小脑子'一次只能记住一定量的东西，超出的就忘了。我们大人也有这个问题啊，只不过我们的容量比 AI 大一点。」

</ReviewBox>
