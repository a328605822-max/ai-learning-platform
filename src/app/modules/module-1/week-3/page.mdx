## 1. 核心概念讲解

### 1.1 System Prompt：AI 的「人设说明书」

System Prompt 是你给 AI 设定的「角色」和「行为准则」。它不是用户能看到的东西，但它是影响 AI 行为的最重要手段。

**好的 System Prompt vs 坏的 System Prompt：**

```
❌ 坏的 System Prompt（太模糊）：
「帮用户回答问题」

✅ 好的 System Prompt（具体、有场景、有边界）：
「你是一个有 10 年经验的 Python 编程导师。你的教学风格是：
1. 用通俗的类比解释概念，避免术语轰炸
2. 每个概念配一个可运行的代码示例
3. 当学生困惑时，先问他们'你觉得是为什么'再给出答案
4. 如果问题超出编程范围，礼貌地引导回编程话题
5. 使用中文回复，代码注释也用中文」
```

<TipBox title="System Prompt 设计原则">
  1. **角色具体化**：不要说「你是一个助手」，说「你是一个有 5 年经验的新媒体运营专家」
  2. **行为边界清晰**：告诉 AI 什么可以做什么不可以做
  3. **输出格式指定**：需要什么格式的输出？多长？什么语气？
  4. **不断迭代**：先用，发现 AI 表现不好就调整 System Prompt
</TipBox>

### 1.2 对话状态管理

在 API 调用模式下，AI 本身没有「记忆」——每次调用都是独立的。对话历史需要由你的代码来维护。

```
用户："我叫小王"  →  你的代码保存：{role: "user", content: "我叫小王"}
AI："你好小王！"  →  你的代码保存：{role: "assistant", content: "你好小王！"}
用户："我叫什么？" →  你的代码把上面两条历史 + 新问题一起发给 API
AI："你叫小王！"  →  AI 能回答出来，因为它看到了历史记录
```

**这就是为什么你需要手动管理对话历史。** 如果一个对话已经很长，超出了上下文窗口，你需要智能地裁剪——保留最近的消息 + System Prompt + 重要的早期消息。

## 2. 动手实践

### 2.1 完成完整的 AI 聊天应用

在上周的 `app.py` 基础上，接入真实的 API。以下是完整代码：

```python
import streamlit as st
import os
from dotenv import load_dotenv
from anthropic import Anthropic

load_dotenv()

# 页面配置
st.set_page_config(page_title="我的 AI 助手", page_icon="🤖")
st.title("🤖 我的 AI 助手")

# ---- API 客户端 ----
client = Anthropic(api_key=os.getenv("CLAUDE_API_KEY"))

# ---- 侧边栏 ----
with st.sidebar:
    st.header("⚙️ 参数设置")
    
    # System Prompt
    system_prompt = st.text_area(
        "System Prompt（AI 的人设）",
        value="你是一个友好的 AI 助手，乐于帮助用户解决问题。",
        height=120
    )
    
    # AI 服务商选择
    provider = st.selectbox(
        "AI 服务商",
        options=["DeepSeek（国内推荐）", "智谱 GLM（免费额度）", "OpenAI", "Claude"],
        index=0,
        help="国内用户推荐 DeepSeek 或智谱 GLM，无需代理"
    )

    # 根据服务商显示对应模型
    model_map = {
        "DeepSeek（国内推荐）": ["deepseek-chat", "deepseek-reasoner"],
        "智谱 GLM（免费额度）": ["glm-4-flash", "glm-4-plus"],
        "OpenAI": ["gpt-4o-mini", "gpt-4o"],
        "Claude": ["claude-haiku-4-5-20251001", "claude-sonnet-4-6-20250514"],
    }
    model = st.selectbox("模型", options=model_map[provider])

    # Temperature
    temperature = st.slider("Temperature", 0.0, 1.0, 0.7, 0.1)

<ImageCard
  src="/images/concepts/temperature.png"
  alt="Temperature 效果谱系"
  caption="调这个滑块就像调收音机——往左拧，AI 像个老学究，每句话都一丝不苟（temperature=0）；往右拧，AI 开始放飞自我，天马行空什么话都敢说（temperature=1）。日常使用推荐 0.3-0.7 之间，既有料又不离谱。"
/>
    
    # Max tokens
    max_tokens = st.slider("最大回复长度", 50, 2000, 500, 50)
    
    st.divider()
    
    # 清除对话
    if st.button("🗑️ 清除对话历史"):
        st.session_state.messages = []
        st.rerun()
    
    # Token 计数和费用估算
    st.divider()
    st.caption("💰 费用估算（非常粗略）")
    
    # 模型价格（每 1M tokens，价格随时变动，以官网为准）
    prices = {
        "deepseek-chat": {"input": 0.14, "output": 0.28},
        "deepseek-reasoner": {"input": 0.55, "output": 2.19},
        "glm-4-flash": {"input": 0, "output": 0},
        "glm-4-plus": {"input": 0.7, "output": 0.7},
        "gpt-4o-mini": {"input": 0.15, "output": 0.60},
        "gpt-4o": {"input": 2.50, "output": 10.0},
        "claude-haiku-4-5-20251001": {"input": 0.25, "output": 1.25},
        "claude-sonnet-4-6-20250514": {"input": 3.0, "output": 15.0},
    }
    model_price = prices.get(model, {"input": 0, "output": 0})

# ---- 对话历史 ----
if "messages" not in st.session_state:
    st.session_state.messages = []

# 显示历史消息
for msg in st.session_state.messages:
    with st.chat_message(msg["role"]):
        st.markdown(msg["content"])

# ---- 用户输入处理 ----
if prompt := st.chat_input("输入你的问题..."):
    # 显示用户消息
    st.session_state.messages.append({"role": "user", "content": prompt})
    with st.chat_message("user"):
        st.markdown(prompt)
    
    # 构建 API 请求的 messages
    # System prompt 作为第一条 system 消息
    api_messages = []
    for msg in st.session_state.messages:
        api_messages.append({"role": msg["role"], "content": msg["content"]})
    
    # 调用 API
    with st.chat_message("assistant"):
        with st.spinner("思考中..."):
            try:
                response = client.messages.create(
                    model=model,
                    max_tokens=max_tokens,
                    temperature=temperature,
                    system=system_prompt,
                    messages=api_messages
                )
                
                ai_reply = response.content[0].text
                st.markdown(ai_reply)
                
                # 显示 token 用量
                usage = response.usage
                input_tokens = usage.input_tokens
                output_tokens = usage.output_tokens
                
                price = prices[model]
                cost = (input_tokens * price["input"] + output_tokens * price["output"]) / 1_000_000
                
                st.caption(
                    f"📊 输入：{input_tokens} tokens · "
                    f"输出：{output_tokens} tokens · "
                    f"本次费用：${cost:.6f}"
                )
                
                # 保存 AI 回复
                st.session_state.messages.append({"role": "assistant", "content": ai_reply})
                
            except Exception as e:
                st.error(f"出错了：{e}")
                st.caption("检查 API Key 是否正确、账户余额是否充足")
```

```bash

```

**步骤 2**：在 Streamlit Cloud 部署

1. 访问 [share.streamlit.io](https://share.streamlit.io)
2. 用 GitHub 账号登录
3. 点击「New app」
4. 选择你的仓库和 `app.py`
5. 点击「Deploy」

**步骤 3**：配置环境变量

在 Streamlit Cloud 的 App Settings → Secrets 中添加：

```toml
CLAUDE_API_KEY = "sk-ant-xxx-你的key"
```

代码里把 `load_dotenv()` 替换为从 `st.secrets` 读取：

```python
# 修改 app.py 的开头部分
api_key = os.getenv("CLAUDE_API_KEY") or st.secrets.get("CLAUDE_API_KEY")
if not api_key:
    st.error("请设置 CLAUDE_API_KEY")
    st.stop()

client = Anthropic(api_key=api_key)
```

<WarningBox title="安全提醒">
  **绝对不要把 `.env` 文件提交到 GitHub！** 在 `.gitignore` 中添加：
  ```
  .env
  venv/
  __pycache__/
  .streamlit/secrets.toml
  ```
  
  如果 API Key 不小心泄露了，立即去 API 提供商的后台删除并重新生成。
</WarningBox>
