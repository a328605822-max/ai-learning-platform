## 2. 需求验证：在写代码之前

### 用 AI 做竞品分析

```markdown
# AI 辅助竞品分析 Prompt 模板

我想做一个 [产品简述，一句话]，面向 [目标用户]。

请帮我：
1. 搜索 3-5 个直接竞品或间接竞品，列出名称和核心功能
2. 分析每个竞品的定价策略
3. 找出它们共同缺失的功能（可能是你的机会点）
4. 评估这个市场的规模和付费意愿

输出格式：表格+简短分析
```

### 用 AI 做用户调研

在写一行代码之前，先找 5 个潜在用户聊聊：

```
不需要做问卷调查。直接在微信/论坛/社群里找目标用户，私聊问：

1. 「你平时做 [某件事] 最大痛点是什么？」
2. 「你现在怎么解决这个问题？」
3. 「如果有人帮你自动完成，你愿意付多少钱？」
4. 「你觉得什么样的方案是「好的」？」

5 个私聊 > 50 份问卷。重点是听「真实需求」不是「礼貌回答」。
```

<WarningBox title="如果用户说'挺好的'">
  「挺好的」= 不会付钱。只有当用户说「我也遇到这个问题！你做好了发我！」的时候，才是真正的需求信号。
</WarningBox>

## 3. 三视角复盘

<ReviewPanel type="principle">
产品方向选择 = 技术可行性 × 市场需求 × 你的优势。技术可行性你已经会判断（前四个模块学到的），市场需求靠竞品分析+用户调研，你的优势是最终的差异化来源。
</ReviewPanel>

<ReviewPanel type="money">
不要做「通用的 AI 聊天工具」（ChatGPT 免费且更好）。做「特定场景的 AI 工具」——写小红书的、写周报的、整理发票的。场景越具体，用户越愿意付费。
</ReviewPanel>

<ReviewPanel type="education">
教孩子「在动手做任何事之前，先想清楚：谁需要这个？他们现在怎么解决的？我做的东西怎么比现有方案更好？」这不是 AI 知识——这是产品思维。AI 时代更需要这个。
</ReviewPanel>
