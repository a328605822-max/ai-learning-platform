### 部署检查清单

```markdown
□ 环境变量全部配置（API Key 等）
□ .env 文件在 .gitignore 中
□ API Key 没有硬编码在代码里
□ CORS 配置正确（如果前后端分离）
□ 基本错误页面（500/404）
□ 日志输出正常
□ 从公网能正常访问
□ 手机端也能打开
```

### 环境变量管理

```python
# 正确的方式：从环境变量读取
import os
api_key = os.getenv("CLAUDE_API_KEY")

# 部署时在平台后台设置环境变量（不要写在代码里！）
# Streamlit Cloud: Settings → Secrets
# Vercel: Settings → Environment Variables
# Railway: Variables
```

<WarningBox title="安全红线">
  1. API Key 绝不出现在 GitHub 上
  2. 每个部署平台的 Secret 都是独立配置的
  3. 如果 Key 意外泄露了——立刻在 API 后台 revoke 并重新生成
  4. 生产环境用只读权限的 API Key（最小权限原则）
</WarningBox>
